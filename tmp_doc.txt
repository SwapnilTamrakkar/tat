1. Product Overview

The Clock Rule Engine is a no-code, browser-based configuration application that allows business users — such as operations managers, configuration analysts, and compliance officers — to define, manage, and test time-tracking rules for case management workflows.

A rule specifies: when a case matches certain criteria (case type, service type, request type, etc.), start, stop, pause, or extend a named clock with a specific duration, threshold alerts, holiday and weekend exclusions, and chaining behavior.

1.2 Data Foundation (from TAT Rules Matrix)

The following session/request type combinations drive rule configuration and must be fully supported by the UI:

IP (Inpatient): Inpatient Admission, Inpatient Psychiatric, Intensive Rehabilitation, MH Extended Bed ≥45 Day

Standard requests: Admission, Prior Auth, Retrospective, Reconsideration → 7-Day TAT / 2-Day Yellow / 1-Day Red / Calendar Days / Next Day Clock / Secondary Clock: YES, 10 Days

Expedited requests → 72-Hour TAT / 36-Hour Yellow / 24-Hour Red / Flat Hours / Same Day / Secondary Clock: NO

OP (Outpatient): Ambulance, Chiropractic, CMOE, Dental-Office, DME, EDBI, Hearing Aids, Home Care Nursing, Home Health, Imaging Studies, Laboratory, Mental Health Services, Occupational Therapy, Physical Therapy, Physician Services, Prosthetics, Substance Abuse, Vision Care

Standard: 7-Day TAT / 2-Day Yellow / 1-Day Red / Calendar Days / Next Day / Secondary Clock: YES, 10 Days

Physician Services Expedited (OP): 72-Hour TAT / 36-Hour Yellow / 18-Hour Red / Flat Hours / Same Day / Secondary Clock: NO, 10 Days

Primary Clock Pause/Resume: triggered by Line Status = Pending + Certified Reason = Additional Info pend + Task Name = Request Additional Info + Task Status = Open.

6. Glossary

Term

Definition

TAT

Turnaround Time — the maximum allowed duration for a case or request to be processed.

Primary Clock

The main compliance clock that runs for a case. A case may have one primary clock.

Secondary Clock

An additional clock that runs concurrently with or after the primary clock, typically triggered by a pause/pend event.

Clock Chaining

The automatic triggering of one clock event by another (e.g., pausing the primary clock starts the secondary clock).

Calendar Days

Counting method that includes all 7 days of the week regardless of holidays or weekends.

Flat Hours

Counting method using absolute hours (24-hour days), not restricted to business hours.

Business Hours

Counting method that only counts hours within the tenant's configured work schedule.

Same Day

The clock starts on the same day as the trigger event.

Next Day

The clock starts on the business/calendar day following the trigger event.

Yellow Alert

A threshold warning indicating a case is approaching its TAT deadline (configurable interval before deadline).

Red Alert

A threshold indicating a case is critically close to or has exceeded its TAT deadline.

IP

Inpatient — case type indicating the patient is admitted to a facility.

OP

Outpatient — case type indicating the patient is treated without admission.

Pause Condition

A set of field=value conditions that, when all true simultaneously, pause the running clock.

Resume Condition

A set of field=value conditions that, when all true simultaneously, resume a paused clock.

Tenant

An isolated client organization within the multi-tenant application, with its own rules, schedules, and holidays.

Draft

Rule lifecycle status: created but not yet activated. Does not affect live cases.

Active

Rule lifecycle status: the rule is evaluated against incoming cases.

Archived

Rule lifecycle status: permanently inactive and hidden from default views. Cannot be re-activated.
